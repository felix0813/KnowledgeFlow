## 功能列表（PRD 级别）

### 0. 基础与通用

* 本地数据存储（离线可用）
* 首次引导（权限/使用说明）
* 设置页（通知开关、每日任务数量、默认标签等）

### 1. 稍后读（收藏与阅读）

**1.1 收集入口**

* 系统分享保存（网页链接/文本）
* 从剪贴板一键保存（可选）
* 手动新增（粘贴链接/输入文本）

**1.2 收藏条目管理**

* 列表：未读/已读/归档筛选
* 标签：添加/删除/筛选
* 搜索：按标题/摘要/URL 搜索
* 详情：打开原文（内置 WebView 或外部浏览器）
* 状态流转：未读 → 在读 → 已读 / 归档
* 收藏条目编辑：标题、备注、标签

**1.3 阅读辅助（可选增强）**

* 阅读模式（简化版：仅 WebView + 字体/夜间）
* 离线缓存（下载网页/仅保存快照文本）

---

### 2. 知识清单（知识点库）

**2.1 知识点 CRUD**

* 新增/编辑/删除知识点
* 字段：标题、内容笔记、标签、优先级、掌握程度（未掌握/模糊/掌握）
* 标记与筛选：按标签、掌握程度、优先级过滤

**2.2 知识点与资料关联**

* 从收藏条目“一键生成知识点”（带入标题/选中文本）
* 一个知识点可关联多篇资料；一篇资料也可关联多个知识点
* 在知识点详情里查看关联资料；在资料详情里查看关联知识点

**2.3 面试模式（建议加，价值高）**

* 1 分钟回答字段：`oneMinuteAnswer`
* 标记：是否“能讲清楚”（面试准备用）

---

### 3. 每日学习任务（任务系统）

**3.1 今日任务清单**

* 今日任务列表：勾选完成/撤销
* 任务类型：阅读（资料）、复习（知识点）、总结（输出）
* 任务编辑：调整预计时长、备注、排序

**3.2 自动生成任务（核心增强）**

* 每日自动生成 N 个任务（可配置）

    * 从未读资料中选 1 个
    * 从“模糊/高优先级”知识点中选 2~3 个
    * 可选：加入“1 分钟复述”任务
* 生成策略：避免重复、优先最近未复习/高优先级

**3.3 提醒与复盘**

* 每日固定时间提醒：未完成任务数量
* 简单统计：连续学习天数、完成率（按周/月）
* “薄弱项”聚合：模糊且高频出现的标签/主题

---

### 4. 标签与分类（横向能力）

* 标签管理：新增/重命名/删除/合并（可选）
* 推荐标签（可选）：根据标题/内容简单规则推荐

---

### 5. 搜索与组织（增强项）

* 全文搜索：知识点内容、资料摘要
* 快速过滤：最近新增/最近复习/高优先级
* 归档与回收站（可选）

## 范围分期建议（便于推进）

* **MVP（第 1 期）**：分享保存 → 资料列表/阅读 → 知识点 CRUD → 关联 → 今日任务手动维护

# 技术方案（工程视角）

## 1) 架构与状态管理（现代）

* **MVVM 或 MVI**

    * UI：Compose/或 XML
    * ViewModel：StateFlow/Flow
    * Domain：UseCase（可选但面试好讲）
    * Data：Repository（单一数据源/离线优先）
* **Kotlin Coroutines + Flow**：异步与响应式数据流

## 2) 数据存储与查询

* **Room**

    * 表：Items / KnowledgePoints / Tags / CrossRef / Tasks
    * 多对多：CrossRef（知识点↔资料）
    * 索引：按状态、日期、标签索引
* **FTS（Room FTS4/FTS5）**（第 2 期）

    * 支持全文搜索知识点内容/资料摘要
* **DataStore（Preferences）**

    * 每日任务数量、提醒时间、默认筛选等配置

## 3) 系统集成（会用到的“旧 API / 常用系统能力”）

* **分享接收**：`Intent.ACTION_SEND`、`ACTION_PROCESS_TEXT`

    * 解析分享内容（text/plain、可能有 URL）
* **WebView**：阅读原文（依旧常见）
* **Notification**（通知渠道、点击跳转）

    * 每日提醒未完成任务
    * 从通知直达“今日任务”
* **WorkManager**

    * 每日自动生成任务（PeriodicWork）
    * 定时检查与补偿（例如错过生成时机）
* **ClipboardManager（可选）**

    * “保存剪贴板链接/文本”入口
* **App Shortcuts（可选）**

    * 桌面长按：新增知识点/保存剪贴板

> 这部分就是你要的“现代项目里还能练旧 API”的核心区域。

## 4) 列表与性能（中高级常考点）

* **Paging 3**：资料列表、知识点列表（数据量上来后很值）
* Compose 的 Lazy 列表性能优化（key、derivedStateOf、分页）
* Room 查询优化：索引、分页查询、避免 N+1

## 5) 依赖注入与模块化（加分项）

* **Hilt（或 Koin）**：注入 DAO/Repo/UseCase
* 模块划分（建议）

    * `core`（utils、基础组件）
    * `data`（Room、Repo）
    * `domain`（UseCase、模型）
    * `feature_inbox` / `feature_knowledge` / `feature_tasks`

## 6) 测试（面试加分，但可后置）

* 单元测试：解析分享文本→提取 URL、任务生成策略
* Room in-memory 测试
* ViewModel 状态测试（Turbine）
